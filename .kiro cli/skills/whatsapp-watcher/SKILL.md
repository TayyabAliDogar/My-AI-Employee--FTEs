# WhatsApp Watcher

Monitors WhatsApp Web for new messages and creates actionable tasks.

## Prerequisites

1. WhatsApp account with Web access enabled
2. Playwright MCP server (for browser automation)
3. QR code authentication capability

## Setup

### 1. Install Dependencies

```bash
# Playwright is already installed via browsing-with-playwright skill
# No additional dependencies needed
```

### 2. Configure Watcher

Create `scripts/whatsapp-config.json`:

```json
{
  "vault_path": "../../Vault",
  "check_interval": 30,
  "priority_contacts": [
    "Client Name",
    "Boss Name",
    "Important Contact"
  ],
  "ignore_groups": [
    "Family Group",
    "Random Group"
  ],
  "keywords": {
    "urgent": ["urgent", "asap", "emergency", "critical"],
    "business": ["invoice", "payment", "contract", "proposal"]
  }
}
```

## Quick Reference

### Start WhatsApp Watcher

```bash
# First time: Authenticate with QR code
python3 scripts/whatsapp-watcher.py --authenticate

# Normal operation
python3 scripts/whatsapp-watcher.py --vault-path "../../Vault"

# Background mode
nohup python3 scripts/whatsapp-watcher.py &
```

### Stop Watcher

```bash
pkill -f "whatsapp-watcher.py"
```

## How It Works

1. **Connect**: Opens WhatsApp Web via Playwright
2. **Authenticate**: Scans QR code (first time only)
3. **Monitor**: Checks for unread messages every 30 seconds
4. **Categorize**: Analyzes message content and sender
5. **Create Action**: Generates task file in `Vault/Needs_Action/`
6. **Mark Read**: Optionally marks message as read

## Authentication

### First Time Setup

### Session Management

- Session stored in `scripts/whatsapp-session/`
- Valid for 30 days
- Re-authenticate if session expires

```markdown
- [ ] Review request
- [ ] Prepare proposal

## Metadata

- Message ID: 3EB0ABCD1234567890
- Processed: 2026-03-17 10:32:00

---

*Auto-generated by WhatsApp Watcher*
```

## Message Categorization

### Priority Levels

| Priority | Criteria |
|----------|----------|
| HIGH | From priority contact OR contains urgent keywords |
| MEDIUM | Business-related keywords OR individual chat |
| LOW | Group messages OR non-urgent content |

### Categories

- **client**: Messages from clients or customers
- **team**: Internal team communication
- **urgent**: Time-sensitive requests
- **business**: Business transactions, invoices, contracts
- **general**: Other messages

## Configuration Options

| Option | Default | Description |
|--------|---------|-------------|
| check_interval | 30 | Seconds between checks |
| auto_mark_read | false | Mark messages as read after processing |
| save_media | false | Download and save media attachments |
| max_messages_per_check | 20 | Max messages to process per cycle |

## Security & Privacy

### Data Handling

- Messages stored locally in Obsidian vault
- No cloud storage or external APIs
- WhatsApp session encrypted by Playwright

### Best Practices

1. **Secure Session**: Keep `whatsapp-session/` folder secure
2. **Regular Re-auth**: Re-authenticate every 30 days
3. **Logout When Done**: Close session if not using
4. **Monitor Access**: Check WhatsApp Web sessions in app

## Troubleshooting

| Issue | Solution |
|-------|----------|
| QR code not appearing | Restart Playwright server and try again |
| Session expired | Run with --authenticate flag |
| Messages not detected | Check if WhatsApp Web is loaded properly |
| Rate limiting | Increase check_interval to 60+ seconds |
| Browser crashes | Restart Playwright server |

## Limitations

- Requires WhatsApp Web to be accessible
- Cannot send messages (read-only monitoring)
- Group messages may have limited context
- Media files require additional handling

## Example Workflow

1. Client sends WhatsApp message: "Need invoice for last month"
2. Watcher detects message, categorizes as "business" + "high priority"
3. Creates action file in `Vault/Needs_Action/`
4. Plan Creator includes it in next PLAN.md
5. Claude processes request, generates invoice
6. Human approves invoice
7. Response sent back to client
